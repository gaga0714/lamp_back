# 管理员端功能精简方案

## 1. 文档说明

本文件保留管理员端精简的设计背景，但内容已整理为当前落地现状。

当前 `admin` 角色已经重新定义为“实验室管理员”，只保留实验室相关后台能力。

## 2. 当前管理员职责

### 保留功能
- 工作台
- 实验室列表
- 预约审批
- 实验室管理
- 实验室使用统计
- 个人资料
- 修改密码

### 已移除功能
- 课程签到
- 课程考勤
- 课程请假
- 我的请假
- 我的预约
- 课表管理
- 考勤总览
- 用户管理

## 3. 当前前端现状

### 3.1 菜单

当前管理员前端菜单应为：
- `工作台`
- `实验室列表`
- `预约审批`
- `实验室使用统计`
- `实验室管理`
- `个人资料`

说明：
- `lab/my-bookings` 当前仅对 `student`、`teacher` 开放
- `attendance/manage`、`course/manage`、`admin/users` 已不再挂入当前管理员菜单

### 3.2 工作台快捷入口

当前管理员快捷入口应为：
- `实验室列表`
- `预约审批`
- `实验室管理`
- `实验室使用统计`

不再显示：
- `课表管理`
- `考勤总览`
- `我的预约`

## 4. 当前后端现状

### 4.1 管理员保留的实验室接口

- `GET /api/lab/booking/approve/list`
- `PUT /api/lab/booking/{id}/approve`
- `GET /api/lab/usage/stats`
- `GET /api/lab/manage/list`
- `POST /api/lab/manage`
- `PUT /api/lab/manage`
- `DELETE /api/lab/manage/{id}`

### 4.2 已从管理员移除的接口

- `GET /api/course/manage/list`
- `GET /api/course/attendance/admin/list`
- `GET /api/attendance/manage`
- `GET/POST/PUT/DELETE /api/admin/users`

这些接口当前即使直接访问，也应返回无权限。

## 5. 当前首页卡片

管理员工作台首页卡片已经改为实验室视角：
- `实验室总数`
- `本月实验室真实使用率`
- `待审批预约`
- `今日实验室预约数`

不再展示：
- `今日课程考勤`
- `用户总数`

## 6. 验证标准

### 管理员登录后
- 看不到 `我的预约`
- 看不到 `课表管理`
- 看不到 `考勤总览`
- 看不到 `用户管理`
- 只能进入实验室相关后台页面

### 学生 / 教师登录后
- 学生、教师仍然可以正常使用 `我的预约`
- 教师仍可使用课程考勤管理与请假审批
- 实验室管理、预约审批、实验室统计不会暴露给非管理员

## 7. 结论

管理员端当前已经完成从“泛后台管理员”向“实验室管理员”的收敛，职责边界为：
- 不参与课程与请假业务
- 不参与账号与用户维护
- 只负责实验室资源、预约审批与实验室使用统计
