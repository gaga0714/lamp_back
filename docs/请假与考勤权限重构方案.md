# 请假与考勤权限重构方案

## 1. 文档说明

本文件保留“考勤与请假角色重构”的设计背景，但内容已按当前代码现状整理。

当前系统中：
- `student` 负责课程签到、课程考勤、课程请假
- `teacher` 负责按授课课程审批请假并管理课程考勤
- `admin` 为实验室管理员，不再参与考勤与请假模块

## 2. 当前角色边界

### 2.1 学生

可访问：
- `POST /api/attendance/check-in`
- `GET /api/attendance/records`
- `POST /api/attendance/leave`
- `GET /api/attendance/leave/list`
- `PUT /api/attendance/leave/{id}/cancel`

前端菜单：
- `课程签到`
- `课程考勤`
- `课程请假`
- `我的请假`

### 2.2 教师

可访问：
- `GET /api/attendance/leave/pending`
- `PUT /api/attendance/leave/{id}/approve`
- `GET /api/course/teacher/schedule`
- `GET /api/course/attendance/teacher/list`
- `GET /api/course/{id}/students`

前端菜单：
- `我的授课`
- `课程考勤管理`
- `请假审批`

### 2.3 管理员

当前不参与考勤与请假业务流。

管理员应满足：
- 看不到课程签到、课程考勤、课程请假、我的请假、请假审批、课程考勤管理
- 直接访问相关接口时返回无权限

## 3. 当前实现特点

### 3.1 请假已与课程绑定

当前 `leave_apply` 已扩展为课程请假模型，核心字段包括：
- `course_id`
- `course_date`

这意味着当前请假不是纯通用假条，而是“针对某门课某一天”的请假申请。

### 3.2 教师按授课课程审批

系统当前不是“任意教师审批任意学生请假”，而是：
- 教师只能看到自己授课课程相关的待审批请假
- 教师审批也会校验课程归属

### 3.3 课程考勤主链路

当前课程考勤核心围绕以下模型运行：
- `course`
- `course_student`
- `course_attendance`
- `leave_apply`

`attendance_record` 仍保留在项目中，但已不是课程签到主流程的核心表。

## 4. 当前前后端入口

### 前端路由

当前考勤与请假相关路由为：

| 路由 | 角色 |
| --- | --- |
| `/attendance/check-in` | `student` |
| `/attendance/records` | `student` |
| `/attendance/leave/apply` | `student` |
| `/attendance/leave/list` | `student` |
| `/attendance/leave/approve` | `teacher` |
| `/attendance/course/manage` | `teacher` |

当前已经不存在面向 `admin` 的 `考勤管理/考勤总览` 入口。

### 后端接口

当前考勤与请假相关核心接口为：

| 接口 | 角色 | 说明 |
| --- | --- | --- |
| `POST /api/attendance/check-in` | `student` | 学生课程签到 |
| `GET /api/attendance/records` | `student` | 学生课程考勤记录 |
| `POST /api/attendance/leave` | `student` | 发起课程请假 |
| `GET /api/attendance/leave/list` | `student` | 我的请假列表 |
| `PUT /api/attendance/leave/{id}/cancel` | `student` | 撤销请假 |
| `GET /api/attendance/leave/pending` | `teacher` | 教师待审批请假 |
| `PUT /api/attendance/leave/{id}/approve` | `teacher` | 教师审批请假 |
| `GET /api/course/attendance/teacher/list` | `teacher` | 教师课程考勤管理 |
| `GET /api/course/{id}/students` | `teacher` | 课程学生与考勤详情 |

`GET /api/attendance/manage` 仍保留在代码中，但当前已对 `admin` 返回无权限，且前端没有对应入口。

## 5. 验证标准

### 学生
- 可正常进入课程签到、课程考勤、课程请假、我的请假
- 只能看到自己的请假与考勤记录

### 教师
- 可正常进入我的授课、课程考勤管理、请假审批
- 只能审批自己授课课程对应的请假

### 管理员
- 无法进入考勤与请假相关页面
- 直接访问相关接口时返回无权限

## 6. 当前结论

本模块当前已经完成从“管理员可查看考勤”到“管理员完全退出考勤/请假流程”的进一步收敛。

当前代码实际语义为：
- 学生负责个人课程考勤与请假
- 教师负责课程考勤管理与请假审批
- 实验室管理员不参与教学考勤事务
