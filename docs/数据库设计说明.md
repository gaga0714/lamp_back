# 研究生考勤及实验室预约系统数据库设计说明

## 1. 概述

- 数据库名：`lamp`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`
- 当前核心表：8 张
- 角色定义：`student`、`teacher`、`admin`

当前项目中的 `admin` 语义已经固定为“实验室管理员”，不再区分 `labAdmin`，也不再承担系统级用户管理职责。

## 2. 当前表结构总览

| 表名 | 说明 |
| --- | --- |
| `sys_user` | 用户表 |
| `attendance_record` | 历史日常考勤记录表 |
| `leave_apply` | 课程请假申请表 |
| `lab` | 实验室表 |
| `lab_booking` | 实验室预约表 |
| `course` | 课程表 |
| `course_student` | 选课关系表 |
| `course_attendance` | 课程考勤表 |

说明：
- 课程考勤主链路已基于 `course / course_student / course_attendance / leave_apply` 运转。
- `attendance_record` 仍保留在实体和建表脚本中，属于历史兼容表。

## 3. 主要关系

```text
sys_user
  ├── 1:N leave_apply
  ├── 1:N lab_booking
  ├── 1:N course_student (student_id)
  ├── 1:N course_attendance (student_id)
  └── 1:N course (teacher_id)

lab
  └── 1:N lab_booking

course
  ├── 1:N course_student
  ├── 1:N course_attendance
  └── 1:N leave_apply (通过 course_id 关联)
```

项目当前未强制建立物理外键，主要通过应用层与索引保证关联与查询性能。

## 4. 各表说明

### 4.1 `sys_user`

用户表，保存登录账号和角色信息。

关键字段：
- `username`：用户名/学号/工号，唯一
- `password`：BCrypt 加密密码
- `name`：姓名
- `phone`、`email`：个人资料
- `role`：`student` / `teacher` / `admin`
- `status`：1 正常，0 禁用

### 4.2 `course`

课程表，描述固定周课表信息。

关键字段：
- `course_code`
- `course_name`
- `teacher_id`
- `semester`
- `term_start_date`
- `weeks`
- `weekday`
- `start_time` / `end_time`
- `location`
- `status`

### 4.3 `course_student`

选课关系表，用于维护“哪些学生要上哪些课”。

关键字段：
- `course_id`
- `student_id`

唯一约束：
- `uk_course_student (course_id, student_id)`

### 4.4 `course_attendance`

课程考勤表，是当前学生课程签到与教师考勤管理的核心表。

关键字段：
- `course_id`
- `student_id`
- `course_date`
- `status`
- `check_in_time`
- `remark`

状态示例：
- `待签到`
- `已签到`
- `迟到`
- `请假`
- `缺勤`

### 4.5 `leave_apply`

课程请假申请表。

关键字段：
- `user_id`
- `type`
- `course_id`
- `course_date`
- `start_time`
- `end_time`
- `reason`
- `status`
- `approve_remark`

当前请假已与课程绑定，教师按授课课程审批，不再是纯通用请假表。

### 4.6 `lab`

实验室表。

关键字段：
- `name`
- `description`
- `location`
- `capacity`
- `equipment_info`
- `status`

状态：
- `available`：可预约
- `maintenance`：维护中

### 4.7 `lab_booking`

实验室预约表。

关键字段：
- `lab_id`
- `user_id`
- `date`
- `slot`
- `purpose`
- `status`
- `approve_remark`
- `check_in_time`
- `check_out_time`

当前状态流转：
- `pending`
- `approved`
- `checked_in`
- `completed`
- `no_show`
- `rejected`
- `cancelled`

### 4.8 `attendance_record`

历史日常考勤记录表，当前不是课程考勤主流程的核心表，但仍保留在项目中。

如果后续确认完全不再使用，可以在新一轮清理中评估是否移除。

## 5. 当前角色与数据库含义

| 角色值 | 含义 |
| --- | --- |
| `student` | 研究生 |
| `teacher` | 教师 |
| `admin` | 实验室管理员 |

当前文档、代码、建表脚本都应以这 3 个角色为准。

## 6. 建库与初始化

推荐直接执行：

```bash
mysql -u root -p < docs/database.sql
```

然后在 `application.yml` 中配置数据源：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/lamp?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的密码
```

项目当前使用：

```yaml
spring:
  jpa:
    hibernate:
      ddl-auto: update
```

因此脚本建表和 JPA 自动更新可以并存。

## 7. 备注

- 默认 `admin` 账号由应用启动时自动创建。
- 样例课程、选课、课程考勤数据也已写入 `docs/database.sql`。
- 若后续需要继续精简数据库文档，建议优先围绕 `course_*`、`leave_apply`、`lab_*` 三条主业务链维护。
