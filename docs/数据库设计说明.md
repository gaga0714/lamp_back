# 研究生考勤及实验室预约系统 - 数据库设计说明

## 1. 概述

- **数据库名**：`lamp`
- **字符集**：`utf8mb4`
- **排序规则**：`utf8mb4_unicode_ci`
- **表数量**：5 张，与后端 JPA 实体一一对应；列名使用下划线（与 Spring JPA 默认命名策略一致）。

## 2. ER 关系简述

```
sys_user (用户)
    ├── 1 : N  attendance_record (考勤记录，按 user_id)
    ├── 1 : N  leave_apply (请假申请，按 user_id)
    └── 1 : N  lab_booking (实验室预约，按 user_id)

lab (实验室)
    └── 1 : N  lab_booking (实验室预约，按 lab_id)
```

- 用户与考勤、请假、预约为一对多；实验室与预约为一对多。
- 未在数据库中建物理外键，由应用层（JPA）维护逻辑关联，便于迁移与扩展。

## 3. 表结构说明

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| **sys_user** | 用户表 | username(唯一), password, name, role, status |
| **attendance_record** | 考勤记录表 | user_id, date(唯一与user_id), check_in_time, check_out_time, status |
| **leave_apply** | 请假申请表 | user_id, type, start_time, end_time, reason, status |
| **lab** | 实验室表 | name, description, capacity, status, open_time |
| **lab_booking** | 实验室预约表 | lab_id, user_id, date, slot, purpose, status |

### 3.1 sys_user（用户表）

| 列名 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键，自增 |
| username | VARCHAR(64) | 用户名/学号/工号，唯一 |
| password | VARCHAR(64) | 密码（BCrypt 加密） |
| name | VARCHAR(32) | 姓名 |
| phone | VARCHAR(20) | 手机号 |
| email | VARCHAR(64) | 邮箱 |
| role | VARCHAR(20) | student / teacher / labAdmin / admin |
| status | INT | 1 正常，0 禁用 |
| create_time, update_time | DATETIME | 创建/更新时间 |

### 3.2 attendance_record（考勤记录表）

| 列名 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 用户 ID |
| date | DATE | 考勤日期，与 user_id 唯一 |
| check_in_time | DATETIME | 签到时间 |
| check_out_time | DATETIME | 签退时间 |
| status | VARCHAR(16) | 正常 / 迟到 / 早退 / 缺勤 |
| create_time | DATETIME | 创建时间 |

**唯一约束**：`(user_id, date)`，保证每人每天一条考勤记录。

### 3.3 leave_apply（请假申请表）

| 列名 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键，自增 |
| user_id | BIGINT | 申请人 ID |
| type | VARCHAR(20) | personal-事假 / sick-病假 / other-其他 |
| start_time, end_time | DATETIME | 开始/结束时间 |
| reason | TEXT | 请假事由 |
| status | VARCHAR(16) | 待审批 / 已通过 / 已驳回 |
| approve_remark | TEXT | 审批备注 |
| create_time, update_time | DATETIME | 创建/更新时间 |

### 3.4 lab（实验室表）

| 列名 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键，自增 |
| name | VARCHAR(64) | 实验室名称 |
| description | TEXT | 描述 |
| capacity | INT | 容量（人） |
| status | VARCHAR(20) | available-可预约 / maintenance-维护中 |
| open_time | VARCHAR(64) | 开放时间描述 |
| create_time, update_time | DATETIME | 创建/更新时间 |

### 3.5 lab_booking（实验室预约表）

| 列名 | 类型 | 说明 |
|-----|------|------|
| id | BIGINT | 主键，自增 |
| lab_id | BIGINT | 实验室 ID |
| user_id | BIGINT | 预约人 ID |
| date | DATE | 预约日期 |
| slot | VARCHAR(32) | 时段，如 08:00-10:00 |
| purpose | TEXT | 用途/人数说明 |
| status | VARCHAR(20) | pending / approved / rejected / used / cancelled |
| approve_remark | TEXT | 审批备注 |
| create_time, update_time | DATETIME | 创建/更新时间 |

**索引**：`(lab_id, date, slot)` 用于查询某实验室某日某时段是否已被预约。

## 4. 使用方式

1. 创建数据库并建表（推荐先执行建表脚本，再让 JPA 使用 `update` 或 `validate`）：
   ```bash
   mysql -u root -p < docs/database.sql
   ```
   或在 MySQL 客户端中执行 `docs/database.sql` 中的语句。

2. 在 `application.yml` 中配置数据源：
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/lamp?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
       username: root
       password: 你的密码
     jpa:
       hibernate:
         ddl-auto: update   # 已有表时用 update 或 validate
   ```

3. 管理员账号由应用启动时自动创建（见 `InitDataRunner`），无需在 SQL 中插入。
