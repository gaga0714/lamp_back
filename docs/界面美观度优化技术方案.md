# 界面美观度优化技术方案

## 1. 目标

根据 `docs/界面的美观度优化需求.md`，本次前端优化聚焦 6 个方向：

1. 侧边栏支持收起/展开
2. 侧边栏标题显示当前登录角色
3. 做 PC / 平板 / 手机多端适配
4. 重做 PC 端工作台，让统计卡片兼具快捷入口能力
5. 重构实验室预约页面：限制过去日期、可视化显示可预约时段、详情页内直接完成预约
6. 美化登录页与注册页

本方案目标是：在不推翻现有业务逻辑的前提下，优先重构布局层与关键页面体验，让系统整体更像一个完整的可演示产品。

## 3. 整体设计原则

### 3.1 视觉原则

- 保持“校园信息系统”风格，简洁、稳定、专业
- 不做过重的炫技动画，避免和业务系统定位冲突
- 统一卡片圆角、阴影、间距、标题层级
- 优先增强信息密度和操作效率，而不是只做颜色变化

### 3.2 技术原则

- 尽量复用现有 Vue 组件与 Element Plus 体系
- 优先通过布局重构和样式抽象实现多端适配
- 不改动核心业务接口语义时，尽量少动后端
- 实验室预约的可视化时段若现有接口不足，再补最小后端接口

## 4. 具体实现方案

## 4.1 侧边栏收起展开

### 目标效果

- PC / 平板横屏：支持手动折叠
- 手机端：侧边栏改为抽屉式弹出
- 收起后仅保留图标
- 展开后显示完整标题和菜单名称

### 主要改动文件

- `D:\bishe\lamp_front\src\layouts\DefaultLayout.vue`
- 可选新增：`D:\bishe\lamp_front\src\stores\ui.js` 或直接在布局组件内管理折叠状态

### 设计方案

1. 在布局中新增状态：
   - `collapsed`
   - `isMobile`
   - `mobileMenuVisible`

2. 通过窗口宽度判断断点：
   - `>= 1200px`：桌面端
   - `768px ~ 1199px`：平板
   - `< 768px`：手机

3. 桌面端：
   - `el-aside` 宽度在 `220px` 与 `64px` 间切换
   - 顶部增加折叠按钮

4. 手机端：
   - 左侧菜单不常驻
   - 改用 `el-drawer` 或固定定位侧栏 + 遮罩
   - 顶部左侧出现汉堡按钮

### 样式要点

- 侧栏宽度变化加过渡动画
- 收起时 logo 文案简化为图标或短字
- 菜单文字隐藏，图标居中

## 4.3 多端适配方案

### 目标范围

支持：
- PC 端（主演示环境）
- 平板端（横屏优先）
- 手机端（基础可用，不追求复杂后台级交互）

### 适配重点页面

1. `DefaultLayout.vue`
2. `DashboardView.vue`
3. `LabDetailView.vue`
4. `LabListView.vue`
5. `MyBookingsView.vue`
6. `LoginView.vue`
7. `RegisterView.vue`

### 实现策略

#### 布局层

- 顶栏高度、左右间距响应式缩放
- 手机端侧栏抽屉化
- 面包屑在小屏缩短或隐藏次级内容

#### 列表 / 表格页

- 对宽表格保留横向滚动
- 手机上减少首屏展示字段
- 表格上方筛选区改为单列堆叠布局

#### 卡片页

- 桌面端 `4 列`
- 平板 `2 列`
- 手机 `1 列`

#### 表单页

- PC 使用左右留白卡片布局
- 手机改为窄边距、满宽输入框、纵向表单

### 建议方式

优先使用：
- SCSS 媒体查询
- `flex` / `grid`
- Element Plus 自带栅格

不建议：
- 引入新的响应式 UI 框架
- 大量 JS 计算布局尺寸

## 4.4 工作台重构

### 当前问题

- 统计卡片和快捷入口割裂
- 视觉上“上面看数据，下面再点按钮”比较重复
- 管理员、教师、学生三种角色工作台差异不够明显

### 目标效果

将卡片改造成“可点击的数据入口卡片”：

- 卡片既展示数据
- 也承担快捷入口
- 鼠标悬停有轻微上浮反馈
- 点击后直接进入对应功能页

### 主要改动文件

- `D:\bishe\lamp_front\src\views\dashboard\DashboardView.vue`

### 重构方案

#### 数据结构调整

前端不再写死四块卡片结构，而是构造一个卡片配置数组：

- `label`
- `value`
- `icon`
- `color`
- `route`
- `description`

根据角色生成不同卡片组。

#### 交互方式

- 整张卡片可点击
- 保留底部少量快捷按钮，或直接移除“快捷入口”模块
- 更推荐直接取消当前单独的按钮区，改为卡片即入口

#### 角色示意

学生：
- 本周课程数 -> 我的课表
- 待签到课程 -> 课程签到
- 我的课程请假 -> 我的请假
- 我的预约 -> 我的预约

教师：
- 本周授课数 -> 我的授课
- 今日授课 -> 我的授课
- 待审批课程请假 -> 请假审批
- 我的预约 -> 我的预约

管理员：
- 实验室总数 -> 实验室管理
- 本月实验室真实使用率 -> 实验室使用统计
- 待审批预约 -> 预约审批
- 今日实验室预约数 -> 实验室列表或预约审批

## 4.5 实验室预约模块重构

这是本次改动里交互变化最大的一项。

### 目标效果

1. 预约日期不能选今天之前
2. 进入实验室详情页时，如果还没选日期，默认展示今天
3. 在详情页可视化展示当前日期所有时段的预约状态
4. 点击某个可预约时段后，下面直接展示预约表单
5. 不再单独跳转到 `LabBookView.vue` 填表

### 推荐页面承载

建议将：

- `LabDetailView.vue`

升级为“实验室详情 + 时段可视化 + 预约表单”的一体化页面。

`LabBookView.vue` 可作为兼容页保留，后续再考虑废弃。

### 前端实现方案

#### Step 1：默认日期逻辑

在 `LabDetailView.vue` 中：
- 若路由没有 `date` 参数，则默认设为今天
- 页面加载时自动查询今天的时段可预约情况

#### Step 2：日期限制

给日期选择器增加：
- `disabled-date`

规则：
- 禁用今天之前的日期

#### Step 3：时段可视化

当前页面只支持选择单个 `slot` 查询，建议改成：

- 固定展示所有时段
- 每个时段渲染成横向条块 / 时间轴卡片

建议展示状态：
- 可预约
- 已占用
- 维护中
- 当前选中

交互：
- 点击“可预约”条块，自动选中该时段
- 选中的时段同步写入预约表单 `form.slot`

#### Step 4：表单下沉到详情页

在时段条形可视化下面直接放预约表单：

- 实验室：只读展示当前实验室
- 日期：当前选中日期
- 时段：当前选中条块自动回填
- 用途/人数：输入
- 提交预约

### 后端支持建议

当前接口不足以直接返回“某天所有时段状态”，建议新增一个轻量接口，例如：

- `GET /lab/{id}/slot-status?date=2026-04-18`

返回结构建议：

```json
{
  "date": "2026-04-18",
  "list": [
    { "slot": "08:00-10:00", "available": true, "statusText": "可预约" },
    { "slot": "10:00-12:00", "available": false, "statusText": "已占用" }
  ]
}
```

这样前端就不需要循环调单时段接口。

### 为什么建议新增接口

如果继续沿用当前单个时段查询：
- 需要前端循环 6 次请求
- 页面加载会慢
- 代码会更绕

新增聚合接口是这次体验重构里性价比最高的后端改动。

## 6. 风险与注意点

### 6.1 布局风险

侧边栏折叠和移动端抽屉会影响：
- 面包屑宽度
- 页面主内容区域宽度
- 表格页可视区域

需要做回归检查。

### 6.2 实验室预约风险

如果把预约表单直接合并进详情页，需要注意：
- 当前从列表进入详情的跳转参数是否兼容
- `LabBookView.vue` 旧入口是否保留
- 预约成功后的跳转逻辑是否改为回到“我的预约”

### 6.3 多端适配边界

本次建议做“响应式后台页面”，不是单独开发移动端专属页面。

也就是说：
- 手机端保证可用
- 平板端保证布局不乱
- PC 端作为主展示端重点打磨

## 9. 结论

本次需求里，最值得重点投入的是三块：

- 主布局响应式重构
- 工作台卡片化重构

它们会直接决定系统演示观感。

登录注册美化和多端适配则建议在统一视觉体系后一起做，避免出现“单页好看但整体不统一”的问题。
